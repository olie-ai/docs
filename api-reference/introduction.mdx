## Visão geral

A API da Olie é uma interface poderosa e flexível que permite integrar e gerenciar processos dentro de uma empresa de forma eficiente e automatizada. A Olie foi projetada para simplificar o gerenciamento de tarefas, fluxos de trabalho e recursos, garantindo que os processos empresariais sejam realizados de maneira mais ágil e controlada.

Para fazer requisições, é necessário autenticar cada pedido com as credenciais da sua aplicação. Se você ainda não tem uma, veja os passos básicos para começar a implementar as nossas soluções.

## Base URL

A URL base para a API da Olie é:

```bash
https://api.olie.ai/api/v1
```

<Tip>
Todas as requisições devem ser feitas utilizando esta URL base como prefixo.
</Tip>

## Autenticação

As solicitações para a API da Olie são autenticadas com um token de acesso da API. O token deve ser passado como um OAuth Bearer token no cabeçalho da requisição:

```bash
curl -H "Authorization: Bearer SEU_TOKEN_AQUI" https://api.olie.ai/api/v1/...
```

<Warning>
Mantenha seu token de acesso em segurança. Nunca compartilhe ou exponha seu token em repositórios públicos ou código cliente.
</Warning>

## Formato das respostas da API

As respostas da API seguem um formato padronizado para facilitar o tratamento dos dados. A seguir, veja exemplos de como interpretar os retornos em diferentes situações.

### Exemplo de resposta de sucesso

```json
{
  "response": true,
  ...
}
```

<Info>
Esta resposta indica que a requisição foi bem-sucedida. Campos adicionais (representados por `...`) podem conter os dados retornados pela operação.
</Info>

### Exemplo de resposta de erro

```json
{
  "response": false,
  "message": "nome_ou_tipo_do_erro",
  "error": "mensagem_detalhada_do_erro"
}
```

<Info>
Essa resposta indica que houve um erro na requisição. A chave `message` descreve o tipo de erro, enquanto `error` detalha a causa.
</Info>
