## Criando um template

Para criar um template no Marketplace, você precisa ter pelo menos uma [empresa construtora](/guides/marketplace/template-builder) com um [snapshot](/guides/marketplace/snapshots) gerado.

<Steps>
<Step title="Selecione a empresa construtora">
  Escolha a empresa construtora que contém o ambiente que você deseja publicar como template.
</Step>

<Step title="Defina o nome do template">
  Escolha um nome claro e descritivo. Esse nome será exibido no grid do Marketplace e é a primeira coisa que os usuários verão.
</Step>

<Step title="Preencha o formulário">
  Complete os campos de metadados do template (detalhados abaixo).
</Step>

<Step title="Selecione o snapshot">
  Escolha qual snapshot será vinculado ao template para submissão.
</Step>

<Step title="Submeta para aprovação">
  Envie o template para revisão do time Olie. Veja mais detalhes no [fluxo de submissão](/guides/marketplace/submission-flow).
</Step>
</Steps>

## Formulário de edição

O formulário de criação e edição do template contém os seguintes campos:

### Descrição curta

Um resumo breve do template que aparece no card do Marketplace. Seja conciso e destaque o principal benefício do template.

### Descrição longa

Uma descrição detalhada do template que aparece na página individual. Suporta **Markdown**, permitindo formatação rica com títulos, listas, negrito e links.

<Tip>
Use a descrição longa para explicar o que o template oferece, o fluxo de trabalho proposto e o que o usuário pode esperar ao implantá-lo. Quanto mais informação, melhor a decisão do usuário.
</Tip>
 
### Categorias

Selecione uma ou mais categorias para o template. As categorias são gerenciadas pelo time Olie e ajudam os usuários a encontrar templates relevantes. Exemplos: DevOps, CRE, Marketing, Vendas. Se você quiser desbravar uma categoria que ainda não existe, entre em contato.

### Imagens

Faça upload de imagens que ilustrem o template. A **primeira imagem** será usada como thumbnail no grid do Marketplace.

### Snapshot

Selecione o snapshot que será vinculado ao template. Esse é o conteúdo que será efetivamente entregue ao usuário quando ele implantar o template.

<Warning>
Certifique-se de selecionar o snapshot correto antes de submeter. O snapshot define exatamente o que o usuário receberá ao implantar o template.
</Warning>
